Файлы для проверки практики 1

Назначение:
- `manual_incident.log` — корректный инцидент. Должен успешно загружаться и проходить проверку.
- `invalid_incident.log` — некорректный инцидент. При загрузке поле идентификатора должно стать `-1`.

Что проверять:
- идентификатор формата две буквы и шесть цифр;
- наименование только из кириллицы;
- краткое описание минимум из 5 слов;
- ущерб: `Отсутствует`, `Незначительный`, `Заметный`, `Существенный`;
- дата и время в формате `дд.мм.гггг чч:мм`;
- имя файла в формате Windows.

Для GUI:
- вкладка `Практика 1 - Инцидент`;
- для быстрой проверки можно указать `manual_incident.log` или `invalid_incident.log`.
