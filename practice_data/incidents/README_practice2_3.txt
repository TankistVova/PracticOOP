Файлы для проверки практик 2 и 3

Назначение:
- `incident01.log` — валидный инцидент. Содержит ключевые слова `загрузка`, `файл`, `EXE`.
- `incident02.log` — валидный инцидент. Подходит для поиска по слову `пароль`.
- `incident03.log` — валидный инцидент без ключевых слов из основного запроса.
- `broken.log` — невалидный инцидент. Должен быть отброшен загрузчиком анализатора.

Что проверять в практике 2:
- загрузчик должен вернуть `3`, так как `broken.log` не должен пройти валидацию;
- в списке наиболее опасных первым должен быть `incident02.log`;
- в списке наименее опасных первым должен быть `incident03.log`.

Что проверять в практике 3:
- запрос `загрузка, файл, EXE` должен найти `incident01.log`;
- запрос `пароль` должен найти `incident02.log`;
- пустой запрос не должен вернуть совпадений.
